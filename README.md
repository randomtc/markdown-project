# Markdown 流式渲染器

基于 React + TypeScript 的 Markdown 渲染器，解决 AI 流式输出时的语法不闭合问题。

**在线演示**: https://randomtc.github.io/markdown-project/

## 核心问题

AI 流式输出时，Markdown 语法可能未闭合（如 `![图片`），直接解析会报错或渲染异常。

## 解决方案：两种流式渲染策略

### 策略一：预处理修复（Streamdown）

**思路**：让不完整语法变完整，再解析渲染

```
Markdown → [修复不完整语法] → 块分割 → React
```

**关键实现**：

1. **块分割**：Marked Lexer 将文档分块
2. **块级 memo**：内容未变的块跳过渲染
3. **语法修复**：`[text]` → `[text](streamdown:incomplete-link)`

**优点**：性能高，适合大文档  
**缺点**：实现复杂，需维护修复规则

---

### 策略二：占位符机制（本项目）

**思路**：未闭合语法用占位符表示，闭合后替换

```
Markdown → [生成占位符] → HTML → React
```

**核心流程**：

```
输入: "![图片](url)"

Step 1: "!"              → 识别为 Image，显示占位符
Step 2-4: "![图片("      → 继续累积
Step 5: "![图片](url)"   → 闭合！渲染真实图片
```

**三层架构**：

| 层级          | 职责                   | 文件              |
| ------------- | ---------------------- | ----------------- |
| 流式预处理    | Token 识别、生成占位符 | `useStreaming.ts` |
| Markdown 解析 | 解析为 HTML            | `Parser.ts`       |
| React 渲染    | HTML 转 React          | `Renderer.ts`     |

**状态机设计**：

```typescript
// 识别 Image 的两种阶段
const REGEX = {
  image: [
    /^!\[[^\]]{0,1000}$/,           // 阶段1: ![ 后无 ]
    /^!\[[^\]]+\]\([^)]{0,1000}$/,  // 阶段2: ]( 后无 )
  ]
};

// 状态流转
Text --"!"--> Image(阶段1) --"](..."--> Image(阶段2) --")"--> Text
```

**缓存结构**：

```typescript
interface StreamCache {
  complete: string; // 已确认的内容（不会变）
  pending: string; // 正在识别的内容（可能变）
  token: TokenType; // 当前状态
}
```

**优点**：实现清晰，易扩展新 Token 类型  
**缺点**：无块级优化，大文档性能一般

## 策略对比

| 维度       | Streamdown       | 本项目            |
| ---------- | ---------------- | ----------------- |
| 核心思想   | 修复不完整语法   | 占位符表示未完成  |
| 性能优化   | ✅ 块级 memo     | ❌ 整体处理       |
| 实现复杂度 | 高               | 中                |
| 扩展性     | 需改修复规则     | Token 识别器模式  |
| 适用场景   | 大文档、高频更新 | AI 流式输出、演示 |

## 技术亮点

1. **增量处理**：只处理新增字符，O(n²) → O(n)
2. **状态机驱动**：Token 识别器模式，易扩展
3. **占位符机制**：未闭合时显示骨架屏，提升体验
4. **组件映射**：`incomplete-image` → 占位组件，`img` → 真实组件
