## 正确的流程

```
输入字符 → 识别Token → 累积到 pending → 生成占位符 → 闭合后提交到 complete
```

## 关键概念澄清

**不是"创建缓存"，而是维护一个状态对象：**

```typescript
interface StreamCache {
  completeMarkdown: string; // 已确认完成的（不会变）
  pending: string; // 正在识别的（可能变）
  token: StreamTokenType; // 当前状态
  processedLength: number; // 处理进度
}
```

**流程示例：**

```
输入: "Hello ![Image](url) World"

Step 1: "H" → pending="H" → token=Text → 直接提交
         complete="H", pending=""

Step 2: "e" → pending="e" → token=Text → 直接提交
         complete="He", pending=""

...

Step 6: "!" → pending="!" → 检测到 ! → token=Image
         complete="Hello ", pending="!"

Step 7: "[" → pending="![" → 还是Image → 继续累积
         complete="Hello ", pending="!["

Step 8-14: "Image]" → pending="![Image]" → 还是Image
         complete="Hello ", pending="![Image]"

Step 15: "(" → pending="![Image](" → 还是Image
         complete="Hello ", pending="![Image]("

Step 16-28: "url)" → pending="![Image](url)" → 闭合！
         检测到 )，正则不匹配 → 提交到 complete
         complete="Hello ![Image](url)", pending=""

Step 29-33: " World" → 逐个提交到 complete
         complete="Hello ![Image](url) World"
```

## 占位符生成时机

```typescript
// 每次处理完新字符后，生成占位符
const placeholder = generatePlaceholder(currentCache, components);

// 输出 = 已完成的内容 + 占位符（如果有）
setOutput(completeMarkdown + (placeholder || ""));
```

**关键点：**

- `completeMarkdown` 是**已完成且不会变**的内容
- `placeholder` 是**当前未完成**内容的占位表示
- 每次新字符输入，都会重新生成占位符

## 一句话总结

> **用 `complete` 存已确认的内容，用 `pending` 累积正在识别的内容，未闭合时显示占位符，闭合后把 `pending` 移到 `complete`，开启下一轮识别。**
